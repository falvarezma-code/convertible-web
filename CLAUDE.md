# Convertible Web

Landing page for Convertible, a privacy-first macOS file converter. Static HTML hosted on Cloudflare Pages. The site root is hand-edited; the blog uses a small build step (see below).

## Writing style

Do not use em dashes (—) in any text content, comments, commit messages, or documentation. Use periods, commas, colons, or parentheses instead, whichever fits the sentence best.

## Blog / Evergreen SEO Pages

The blog is generated by `build-blog.py` (Python 3, stdlib only).

- Templates live in `templates/`: `blog-base.html` (article shell) and `blog-index.html` (index shell).
- Content lives in `blog/<slug>.json`, one file per post.
- Build with: `python3 build-blog.py`
- Output: `<slug>.html` at the repo root for each post, plus a regenerated `blog.html` index.

To add a new post: copy an existing JSON in `blog/`, edit the fields, run the build script. To rebuild everything (after a template tweak), just rerun the script.

URL convention: `convert-{from}-to-{to}-mac.html` for conversion guides, `compress-{format}-mac.html` for compression guides. Flat files at the root, no nested directories.

Each generated article page includes four JSON-LD blocks: `SoftwareApplication`, `BreadcrumbList`, `Article`, `FAQPage`. The build script generates these automatically from the JSON content; do not hand-edit the structured data in the output HTML.

The `blog.html` index is auto-generated and sorted newest first by the `date` field in each JSON. Do not edit it by hand; edit a content file and rebuild.

Design: do not change the article styles to match a per-page taste. The reference design lives in `templates/blog-base.html` and was extracted from the original `convert-mkv-to-mp4-mac.html`. Style changes belong in the template, not in individual pages.

The build script enforces the no-em-dashes rule on every content file and exits non-zero if it finds one.
